# Action Mode Survival AI

行動モード中の「あたまのいいゾンビ」は敵対せず、戦闘モードで使う装備をサバイバル進行によって準備します。

## 採掘

- `ForgeMod.BLOCK_REACH`（既定値はプレイヤーと同じ4.5ブロック）を目から対象ブロック中央までの判定に使います。
- Forgeのサバイバル `FakePlayer` と `ServerPlayerGameMode.destroyBlock` を使うため、硬さ、採掘速度、適正ツール、エンチャント、ステータス効果、耐久消費、ドロップ、経験値、保護Modの破壊イベントがプレイヤー規則を通ります。
- 手持ちと41スロットのインベントリを比較し、対象に対する破壊速度が最も高い適正ツールへ持ち替えます。
- 破壊のひび割れを通常の採掘進捗として同期し、落ちたアイテムを回収します。
- 木材、丸石／深層岩の丸石、石炭、鉄原石を必要数まで探します。適正ツールがない鉱石は破壊しません。

## クラフトと設備

- RecipeManagerに登録された実際のクラフトレシピとIngredientタグを参照し、材料とクラフト残余アイテムを正しく消費します。
- 2x2で作れる板材、棒、作業台から開始し、3x3レシピは到達可能な作業台のそばだけで実行します。
- 作業台とかまどを足場のある隣接位置へ設置します。離れても位置を覚え、必要時に戻ります。
- かまどは実在する `AbstractFurnaceBlockEntity` の入力・燃料・出力スロットを使います。鉄原石を投入し、石炭、木炭、板材、原木を燃料にして通常時間で精錬します。
- 優先装備は木／石／鉄のツール、剣、盾、バケツ、弓、クロスボウ、矢です。生成物がインベントリに収まらない場合は足元へ落とします。

## モード境界

このGoalは行動モードだけで動きます。プレイヤーの視界に入り戦闘モードへ切り替わると直ちに採掘進捗と移動を停止し、攻撃AIへ制御を渡します。行動モードの採掘・クラフト処理自身はプレイヤーをターゲットにしません。

## 再現テスト

`voxpilot/action-survival-progression.json` は、画面外クライアントで資源場を作り、三人称カメラから収集・クラフト・設備設置・精錬を観察するシナリオです。

```powershell
java -jar '..\VoxPilot\VoxPilot.jar' run --project '.' --scenario 'voxpilot\action-survival-progression.json'
```
